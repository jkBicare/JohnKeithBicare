RAFI MFI - One App

Date Started: April 2025

Tools Used:

Description:

OneApp is a project assigned to our team during our internship at RAFI MFI, aimed at developing an internal admin tool. I was in charge of designing the system based on the User Story requirements given by our Product Owner (P.O.). The project consists of three main service pages: Roles and Access, Service Permissions, and Audit Trail. The Roles and Access page allows administrative users to manage access control for different applications based on user roles. This ensures that only authorized users can access specific features or systems. The Service Permissions page also handles application access, but instead of assigning roles, it enables admins to turn services on or off for all users. This gives them direct control over what parts of the application are available. The Audit Trail service acts as a logging system that records any changes or actions taken within the application. It serves as a valuable tool for tracking activities and ensuring accountability. I designed the mockups in Figma, making sure the layout was clear and user-friendly. For development, we used the T3 stack, which gave us hands-on experience in modern full-stack development. OneApp taught us the importance of role-based access, system monitoring, and effective UI/UX design in enterprise-level applications.
